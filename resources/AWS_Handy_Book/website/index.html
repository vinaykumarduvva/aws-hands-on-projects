☁️ Cloud Fundamentals

Cloud Foundations

Core concepts every cloud practitioner must understand before launching a single resource.

🌐

What is AWS?

Amazon Web Services (AWS) is a secure cloud computing platform providing on-demand access to computing power, database storage, content delivery, and other IT infrastructure services over the internet, billed on a pay-as-you-go basis.

💡

On-Demand Self-Service

Provision resources programmatically via the Console, CLI, SDK, or Infrastructure-as-Code — no manual procurement or hardware waiting times required.

💰

Pay-as-you-go Pricing

Pay only for what you consume. No large upfront capital expenditure (CapEx) — shift to operational expenditure (OpEx). Scale costs linearly with usage.

📈

Rapid Elasticity

Scale resources up/down (vertical) or out/in (horizontal) dynamically to meet application demand. AWS handles capacity so you don't over-provision.

🛡️ The Shared Responsibility Model

Security is a joint effort. AWS secures the infrastructure OF the cloud; you secure everything you build IN the cloud.

Responsibility 🔒 AWS (Security OF the Cloud) 👤 Customer (Security IN the Cloud)
Physical Assets Security of hardware, servers, cables, and data centers. N/A (Abstracted away)
Infrastructure Virtualization hypervisor, network hardware, underlying OS. VPC design, firewall rules (Security Groups).
Data Protection Service durability, replication (e.g., S3's 11 nines). Data encryption (in transit/at rest), Bucket Policies, KMS Keys.
Access Control Physical access control to facilities, IAM system uptime. Setting up IAM Users, Groups, Roles, Policies, MFA.
Operating System Hypervisor OS patching. Guest OS configuration (patching EC2 instances).

🕹️ How to Access AWS

🖥️

Management Console

Web-based graphical interface. Great for beginners, visual explorers, and quick configuration changes.

⌨️

AWS CLI

Command-line tool to manage AWS services from your terminal. Supports shell scripting for automation.

🧩

AWS SDKs

APIs for programming languages (Python boto3, Node.js, Java, Go) to build applications that interact with AWS.

📝

Infrastructure as Code

Declarative template-based deployment using CloudFormation or Terraform for safe, repeatable infrastructure.

🌍 Global Infrastructure

Regions & Availability Zones

AWS's global infrastructure of Regions, Availability Zones, and Edge Locations.

🗺️

What is a Region?

A separate physical geographic area containing multiple isolated data centers. Each Region is fully independent and isolated from other Regions.

🏢

Availability Zones (AZs)

One or more discrete data centers within a Region, with independent power, cooling, and networking. AZs are physically separated (miles apart) to limit blast radius.

🌐

Edge Locations

Points of Presence (PoPs) used by CloudFront (CDN) and Route 53 (DNS) to cache content close to end-users globally, reducing latency.

🌍 Global Scope

IAM, Route 53, CloudFront — accessible everywhere, not tied to a single region.

🗺️ Regional Scope

S3 Buckets, VPCs, ALB, DynamoDB Tables — live in a specific region.

📍 Zonal Scope

EC2 Instances, EBS Volumes, Subnets — tied to a specific Availability Zone.

📡 Local Zones

Extensions of a Region that place services close to large population and IT centers.

📶 Wavelength Zones

AWS compute embedded within 5G networks for ultra-low-latency mobile applications.

🏠 Outposts

Hybrid cloud: run native AWS infrastructure on-premises in your own data center.

⚡ High Availability vs. Disaster Recovery

Aspect ⚡ High Availability (HA) 🌪️ Disaster Recovery (DR)
StrategySpan across multiple AZs within one RegionReplicate across multiple Regions globally
Failure TargetData center power/hardware failuresFull-region natural disaster or fiber outage
ComplexityLow-to-moderate; built-in Multi-AZ optionsHigh; requires cross-region replication
LatencyLow (sub-ms synchronous)Higher (asynchronous across continents)
AWS ExamplesRDS Multi-AZ, ASG in multiple subnetsS3 CRR, Route 53 Active-Passive routing

💰 Data Transfer Costs — Rules of Thumb

  • Inbound Data is always FREE (data coming into AWS).
  • Intra-AZ Data is FREE when using private IPs (same AZ communication).
  • Inter-AZ Data is CHARGED ($0.01/GB each direction, different AZs).
  • Outbound Data (to internet) is the MOST EXPENSIVE charge.
